JavaScript组成

JavaScript是一种专为与网页交互而设计的脚本语言，由下列三个不同的部分组成：

  • ECMAScript （核心）
  • DOM （文档对象模型）
  • BOM （浏览器对象模型）

语法

区分大小写

标识符

  • 第一个字符必须是一个字母、下划线( _ )或一个美元符号( $ )
  • 其他字符可以是字母、下划线、美元符号或数字
  • 不能把关键字、保留字、true、false和null用作标识符

注释

  • // 单行注释
  • /**/ 多行（块级）注释

关键字和保留字

关键字：用于表示控制语句的开始或结束，或用于执行特定操作等，不能用作标识符

保留字：将来有可能会被用作关键字

变量

ECMAScript的变量是松散类型的，可以用来保存任何类型的数据
定义变量要使用var操作符

注意：使用var操作符定义的变量将成为定义该变量的作用域中的局部变量

数据类型*

ECMAScript有5种简单数据类型（基本数据类型）：

  • Undefined
  • Null
  • Boolean
  • Number
  • String

1种复杂数据类型：

  • Object

typeof操作符

用于检测给定变量的数据类型，对一个值使用typeof操作符可能返回下列某个字符串：

  • “undefined” ———— 该值未定义
  • “boolean” ———— 该值是布尔值
  • “string” ———— 该值是字符串
  • “number” ———— 该值是数值
  • “object” ———— 该值是对象或null
  • “function” ———— 该值是函数

Undefined类型

该类型只有一个值即undefined，在使用var声明变量但未对其初始化时，这个变量的值就是undefined

Null类型

该类型只有一个值即null，从逻辑角度，null值表示一个空对象指针，所以使用typeof操作符检测null值会返回“object”
实际上，undefined值是派生自null值的，因此ECMA-262规定对它们相等性测试要返回true

Boolean类型

该类型有两个字面值：

  • true
  • false

注意：true和false区分大小写，True和False或其他混合大小写的形式都不是Boolean值，只是标识符

ECMAScript中所有类型的值都有与这两个Boolean值等价的值，可以调用转型函数Boolean()

数据类型 转换为true的值 转换为false的值
Boolean true false
String 任何非空字符串 “”( 空字符串 )
Number 任何非零数字值(包括无穷大) 0和NaN
Object 任何对象 null
Undefined n/a(不适用) undefined

Number类型

多种数值字面量：

  • 十进制整数

  • 八进制（第一位必须是0，然后是八进制数字序列(0~7)，字面值中的数值超出了范围，前导零将被忽略）

  • 十六进制（前两位必须是0x，后跟任何十六进制数字(0~9及A~F)，其中A~F可大写或小写）

注意：进行算术计算时，所有八进制和十六进制表示的数值都将被转换成十进制数值

浮点数值

数值中包含一个小数点，并且小数点后必须至少有一位数字

对于极大或极小的数值可以用e表示法（即科学记数法）表示的浮点数值表示：

注意：浮点数值的最高精度是17位小数，但在进行算术计算时其精确度远远不如整数，例如，0.1加0.2结果不是0.3，而是0.30000000000000004，这个小小的舍入误差会导致无法测试特定的浮点数值

数值范围

由于内存的限制，ECMAScript并不能保存世界上所有的数值

  • 最小数值保存在Number.MIN_VALUE中 ———— 大多数浏览器中这个值是5e-324
  • 最大数值保存在Number.MAX_VALUE中 ———— 大多数浏览器中这个值是1.7976931348623157e+308

如果某次计算结果超出JavaScript数值范围，这个数值将被转换成Infinity（正无穷）或-Infinity（负无穷），该值将无法继续参与下一次的计算，可以使用isFinite()函数确定一个数值是否无穷，参数位于最小与最大数之间会返回true：

NaN

即非数值（Not a Number），用于表示一个本来要返回数值的操作数未返回数值的情况

NaN本身有两个非同寻常的特点：

  • 任何涉及NaN的操作（例如NaN/10）都会返回NaN
  • NaN与任何值都不相等，包括NaN本身

数值转换

非数值转换成数值：

  • Number() ———— 可用于任何数据类型
  • parseInt() ———— 字符串转换成数值
  • parseFloat() ———— 字符串转换成数值

Number()函数转换规则：

  • Boolean值 ———— true转换为1，false转换为0
  • 数字值 ———— 简单的传入和返回
  • null值 ———— 返回0
  • undefined值 ———— 返回NaN
  • 字符串
    • 只包含数字 ———— 转换为十进制数值
    • 包含有效的浮点格式 ———— 转换为对应的浮点数值
    • 字符串中包含有效的十六进制格式 ———— 转换为相同大小的十进制整数值
    • 空字符串 ———— 转换为0
    • 字符串中包含除上述格式之外的字符 ———— 转换为NaN
  • 对象 ———— 调用对象的valueOf()方法，然后依照前面的规则转换返回的值，若值为NaN，调用对象的toString()方法再依照前面的规则转换返回的字符串

parseInt()函数转换规则：

  • 字符串 ———— 忽略字符串前面的空格直至找到第一个非空格字符，第一个字符不是数字字符或者负号会返回NaN；第一个字符是数字字符会继续解析直至遇到非数字字符
  • 浮点数值 ———— 转换为小数点前的整数
  • 不同进制 ———— 可正常解析十六进制，ES3可解析八进制，ES5不可解析八进制，故可为函数提供第二个参数指定基数解除分歧，若参数为16则可以省略字符串前面的“0x”

  • 字符串 ———— 与parseInt()函数一样，不同的是parseFloat()函数还会解析第一个小数点
  • 浮点数值 ———— 识别所有浮点数值格式
  • 不同进制 ———— 会忽略前导零，因此十六进制格式始终被转换为0，只解析十进制，故没有第二个参数指定基数的用法

字符串的特点

字符串是不可变的，一旦创建，值就不能改变
改变某个变量保存的字符串实际上是先销毁原来的字符串，再用包含新值的字符串填充该变量

转换为字符串

把一个值转换为字符串：

  • toString() ———— 除null和undefined值没有该方法，其他数据类型都有该方法；调用数值的toString()方法时可以传递一个参数，即输出数值的基数，传入2则以二进制输出，传入8则以八进制输出
  • String() ———— 将任何值转换成字符串，有toString()方法的调用toString()方法，值是null返回“null”，值是undefined返回“undefined”

Object类型

创建新对象：

Object的每个实例都具有下列属性和方法：

  • constructor：保存着用于创建当前对象的函数
  • hasOwnProperty(propertyName)：用于检查给定的属性在当前对象实例中是否存在（propertyName必须是字符串）
  • isPrototypeOf(object)：用于检查传入的对象是否是当前对象的原型
  • propertyIsEnumerable(propertyName)：用于检查给定的属性是否能够使用for-in语句来枚举
  • toLocaleString()：返回对象的字符串表示，该字符串与执行环境的地区对应
  • toString()：返回对象的字符串表示
  • valueOf()：返回对象的字符串、数值或布尔值表示，通常与toString()方法的返回值相同

递增和递减操作符

递增和递减各有两个版本：

一元加和减操作符

一元加操作符（+）：放在数值前面不会产生影响，放在非数值前面会像Number()转型函数一样先对这个值进行转换
一元减操作符（-）：主要用于表示负数，放在数值前面该值会变成负数，放在非数值前面依旧会先转型然后再变成负数

位操作符

按内存中表示数值的位来操作数值

ECMAScript中的所有数值都以IEEE-754 64位格式存储，位操作符将64位的值转换成32位的整数再将结果转换回64位
对于有符号的整数，前31位表示整数的值，第32位是符号位，0表示正数，1表示负数

  • 正数：纯二进制格式存储，空位填0
  • 负数：二进制补码格式存储
    • 求这个数值绝对值的二进制码
    • 求反码，即0替换1，1替换0
    • 反码加1

注意：在处理有符号整数时，是不能访问位31的

副效应：对特殊的NaN和Infinity值应用位操作时，都会被当成0来处理

布尔操作符

逻辑非

（ ! ）表示逻辑非操作符，可以应用于任何值，无论是什么数据类型都会返回一个布尔值

逻辑非操作符遵循下列规则：

操作数 返回结果
对象 false
“”（空字符串） true
非空字符串 false
0 true
非0数值(包括Infinity) false
null true
NaN true
undefined true

逻辑非操作符也可用于将一个值转换为其对应的布尔值，同时使用两个逻辑非操作符相当于使用Boolean()转型函数

逻辑与

（&&）表示逻辑与操作符，有两个操作数

逻辑与真值表：

第一个操作数 第二个操作数 结果
true true true
true false false
false true false
false false false

若有一个操作数不是布尔值则不一定返回布尔值，此时遵循如下规则：

  • 第一个操作数是对象 ———— 返回第二个操作数
  • 第二个操作数是对象 ———— 第一个操作数求值结果为true时才会返回该对象
  • 两个操作数都是对象 ———— 返回第二个操作数
  • 第一个操作数是null ———— 返回null
  • 第一个操作数是NaN ———— 返回NaN
  • 第一个操作数是undefined ———— 返回undefined

注意：逻辑与操作属于短路操作，第一个操作数能决定结果就不会对第二个操作数求值
